# 自動化規劃（初版 待確認）

## 一句話版

把妳每天重複做的事——尤其是「同一份訂單資料手動打兩次」——用程式幫妳自動完成，從最高效益的工單自動化開始做起。

## 背景和動機

業務助理每天處理 5–20 筆訂單，格式固定、來自老客戶。
最大痛點：收到客戶 Excel 訂單後，要手動重打進工單範本（給工廠），再打一次進公司 ERP。
同樣的資料每天打兩遍，是最浪費時間、也最容易出錯的地方。

---

## 問卷結果整理

根據問卷回覆，九項日常工作的自動化可行性評估如下：

| 工作項目 | 能自動化？ | 省時效益 | 說明 |
|---|---|---|---|
| ① 收訂單 | ✅ 可做 | ★★ | 訂單存進固定資料夾＋按按鈕觸發，後續流程自動接手 |
| ② 輔助報價 | ❌ 不做 | ★ | 大多數客人是老客戶不需報價，效益有限 |
| **③ 製作工單** | **✅ 完全可自動化** | **★★★★** | **本次重點，詳見下方** |
| **④ 登錄 ERP** | **✅ 可自動** | **★★★★** | **本次重點，ERP 支援批次匯入，詳見下方** |
| ⑤ 催交期 | ✅ 可做，待補幾個習慣細節 | ★★ | 自動分階段提醒催貨，取代腦記 |

## 本次重點：③ 製作工單 + ④ 登錄 ERP

這兩項合起來能真正解決「同一份資料打兩次」的問題，是目前效益最高、最值得優先做的部分。

### 怎麼運作

1. 讀取客戶傳來的訂單 Excel
2. 自動填入工單範本 → 存到輸出資料夾，可直接用或寄給工廠
3. 同時產出 ERP 批次匯入用的格式 → 妳去 ERP 點一下匯入即可，不用再手動打第二次

### 預計成果

- 收到訂單 Excel → 一個動作 → 工單自動好 + ERP 匯入檔自動好
- 每天估計可省下 30 分鐘～1 小時（依訂單數量而定）
- 程式只負責「準備好」，最後匯入、確認都還是妳自己按，不會有東西在妳沒看過的情況下就送出

### 動工前需要的三份文件

| # | 文件 | 用途 |
|---|---|---|
| 1 | 客戶訂單 Excel 範本 | 讓程式知道從哪個欄位讀料號、數量、交期（假資料遮掉即可） |
| 2 | 工單 Excel 範本 | 讓程式知道要填進哪些格子（假資料遮掉即可） |
| 3 | ERP 批次匯入範本 | 讓程式產出 ERP 看得懂的格式（ERP 匯入介面點「下載範本」即可拿到） |

拿到這三份後，③④ 就可以直接開始，沒有其他卡點。

## 不包含在這次的範圍

- 輔助報價（ERP 已處理）
- 安排物流（別人負責）
- 開發票（財務負責）
- 整理定期報表（不需要）

## 想請妳協助確認的問題

**工單自動化（③④ 相關）：**
- [ ] 客戶訂單 Excel 範本
- [ ] 工單 Excel 範本
- [ ] 所有客戶的訂單格式都一樣，還是每個客戶長得不同？
- [ ] ERP 裡的品號、客戶代號，跟客戶 Excel 裡的一樣嗎？還是兩邊代碼不同需要對照？

**自動寄 Email 給工廠（尚未決定要不要做）：**
> 若決定做，方式是「程式準備好草稿，妳確認後手動發送」，不會全自動發出。
- [ ] 妳用什麼發 Email？（Outlook 桌面版 / Gmail / 公司信箱）
- [ ] 工廠的 Email 是固定的嗎？還是不同產品對應不同工廠？

**催交期追蹤：**
- [ ] 下單給工廠後，通常幾天內妳會去催第一次？
- [ ] 催了之後，幾天沒回妳會再催一次嗎？
- [ ] 妳用哪個 Email 收每日提醒？

**追蹤付款：**
- [ ] 目前追蹤客戶付款習慣是用紙本還是 Excel／程式？
